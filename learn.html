Tutorial

Introduction

Welcome to VESPY! VESPY is a programming environment on the Web. It enables you to modify or add extra functionality to existing web pages. Here are some examples that enhancements can do:

  • Hide unwanted information such as advertisements.
  • Collect information from multiple web pages, and create custom reports.
  • Add custom filters to any list.
  • Convert plain data tables into interactive calculators.
In this tutorial, you will learn the VESPY's basic usage.







Basic Concepts & UI components

Let's begin with the terminology of VESPY programming.
Try Click the button below to open the practice page. The page will show the VESPY open.
  • The Grid is a canvas you will build enhancements in.
  • The Info pane has the title, description of the current enhancement. You can also save, run, and reset current enhancement there.
  • The Actions pane contains available operations that you can drag to the grid and create a new node. VESPY provides over 30 operations. In the tools panel, you can drag them to the grid. Four colors of the operations represent different roles they have.

Let's look at the enhancement currently shown in the grid. Click a node to see its detail.

  • Enhancement is a unit of VESPY application. An enhancement consists of many nodes.
  • Nodes are placed on the grid UI. Each node contains an operation, a list of values, and is connected to one or multiple input nodes. Based on the connections, nodes will pass values each other, and wil execute its operation.
    • Operation is what this node will do. For example, the operation above multiplies the numbers of input 1 and input 2, and updates the node's value. Parameters such as input 1, *, input 2 can be directly modified.
    • Input nodes are connections to other nodes. By default, a node is connected to its above and left nodes, however, you can change it by pressing and then selecting a node.
    • Values can be set by typing in the text box.

Connecting Nodes

Connections among nodes not just pass the data but also trigger the following nodes. For instance, in the figure below, the order of executing the nodes is represented with red lines.
  1. (0) triggers (1) to set the original numbers.
  2. (2) and (3) are executed to get the count and the sum.
  3. After all, (4) calculates the average using values from (2) and (3).
Three rules about executing nodes are important.
  1. All the nodes to run must be connected to the main Trigger node.
  2. A node awaits until all its preceding nodes are executed.
  3. Trigger nodes awaits until specific event has occured.

2.Extracting Information

Manual
Web elements contain many kinds of attributes. For example, a hyperlink element may contain text, download, visibility, color, background-color, link, and more. To maually extract attributes from elements, follow the steps below.
  1. Drag [Get Attribute] operation from the action panel to the below of the input node.
  2. Change the parameter to the attribute key that you want to extract.

3.Filtering items

Manual
In order to filter items by arbitrary logic, you use [Filter] operation. [Filter] operation needs two inputs, 1) original list of items, 2) boolean values that indicate whether corresponding items will stay in the filtered list. Let's filter a text list by whether they contain "apple". Filtering numbers is the same except you use [Number Test] instead of [String Test].
